# ShopMind 项目亮点 / 创新点

> 与同类电商导购 RAG 方案相比，ShopMind 的三个核心差异化。

---

## 1. 自适应检索：按查询形态"分诊"，而非单一 Hybrid 一刀切

同类方案多用统一 Hybrid 检索应对所有查询，在指名查找（"iPhone 16"）和纯条件筛选（"500 以内的运动鞋"）上既浪费算力、又引入语义噪声。

ShopMind 用一个轻量 Planner 把每条查询分诊到 4 种策略——纯 SQL / ID 直取 / 硬过滤语义 / 纯语义：

- 结构化筛选 → 走 SQL，零 LLM 成本、零向量检索；
- 指名查找 → ID 直取，省掉一整轮向量检索 + rerank，零语义噪声；
- 开放语义 → 才走完整混合检索 + rerank。

**用对的工具做对的查询——更快，也更准。**

---

## 2. 防幻觉是"架构级硬保证"，不是"提示词恳求"

多数 RAG 导购靠 prompt 叮嘱模型"不要编造"，但库里没货时仍会硬凑一个、或把商家描述当参数读出来。

ShopMind 用三道互相独立的结构防线：

1. 硬性条件走 SQL 精确过滤，不交给 LLM 判断；
2. 候选相关性低于阈值就诚实回"没找到符合的"，而非强行返回 top-k；
3. 商品事实只允许引用数据库结构化字段，不读原文凑数。

无幻觉从"概率上尽量做到"变成"结构上不可能违反"。

---

## 3. 双面评论摘要：连"注意点"也替用户讲清楚

同类导购倾向只报喜不报忧，可信度存疑。

ShopMind 在离线阶段对每个商品的评论做正负分层采样、按真实差评占比校准措辞，产出"亮点 + 客观注意点"的两面摘要（评论里没有共性缺点就留空、绝不硬编）。

**不止于"不编造"，而是敢于主动替用户讲出负面。** 让推荐从"看起来可信"变成"经得起追问"。
